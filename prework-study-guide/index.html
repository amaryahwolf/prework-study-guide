HTML

  • Hypertext Markup Language (HTML) creates content and gives content structure with text, images, headers, lists, layout, etc.
  • the head element contains information, or metadata, that helps the browser render the page correctly; the elements included are not visisble when a user looks at a webpage
  • the body element contains the content that is shown to the user; all the elements that we want the user to see and interact with must be placed inside this element
  • declarations are not case-sensitive; HTML elements are case-sensitive
  • an HTML attribute defines additional information about an element; e.g. html lang="en" means the we language for the webpage is english
  • there are lots of metadata tags to learn that u can use inside the head element
  • title element defines the title for a webpage and is important for SEO
  • style element defines the CSS styles associated with a website
  • base element defines the base URL for a webpage
  • link element connects an external resource to the HTML document
  • meta element defines metadata such as the character set, description, keywords, author and viewport
  • script embeds client-side scripts in the HTML document
  • charset is the character set
  • !DOCTYPEhtml tells us the document is written in HTML
  • http-equiv sets an HTTP header for content
  • viewport sets the default view for our webpage
  • see HTML meta tags to learn more about other types of info that can be stored in the meta tag
  • parent/child element - the body element is the parent and 3+ elements nestled inside the body is the child
  • semantic HTML elements: the name of the element describes its content and the role it plays on a webpage
  • div creates a division on the page
  • span creates an inline division
  • the opening tag and closing tag contain the contents of the visible HTML elements; closing tag has a forward slash
  • note: an element refers to the entire element from opening tag to closing tag; a tag refers only to what appears inside the angle brackets
  • h1, h2, h3, h4, h5, h6 are headings; u can only have one h1 on a webpage; this is important for screen readers
  • p is a paragraph or block of text
  • ul creates an unordered list
  • ol creates an ordered list
  • li creates listed items
  • img contains information about images displayed on a webpage and can contain various attributes; e.g. src attribute defines the location of the image & alt attribute contains a text string that describes the appearance and functionality of the image - also important for screen readers and assistvie technologies
  • br creates a line of empty space or a line break between two blocks of content
  • a stands for anchor and creates links to the same webpage or other webpages; href attribute points to the URL for the link
  • section elements are semantic elements that define a section in a document that contains thematic content

CSS

  • Cascading Style Sheets (CSS) styles the content of an application with fonts, colors, spacing, etc.
  • inline CSS: style an element directly in the HTML file; useful for quick and permanent changes but less flexible than external and internal style sheets
  • internal style sheet: contains CSS rules for the webpage in the head section of the HTML file; a single change to the CSS rule will apply to all tagged page elements; uses style element
  • external style sheet: (most common way to style a webpage) allows devs to keep all CSS rules in a separate file, which makes design changes easier; uses link element
  • declaration: contains the CSS property and value of the property
  • margin (property): indicates how much space we want around the outside of an element
  • padding (property): adds space around the content inside an element; increasing padding makes the entire element bigger
  • see the box model for more on margins, borders, padding and content
  • asterisk: wild card symbol: declares that ALL elements on the page will have the same style properties
  • can combine 2 selectors (e.g. header and footer) into a single CSS rule with a comma .. helps us write Do Not Repeat Yourself (DRY) code; can greatly reduce the overall size and complexity of a codebase
  • text-align: positioning property
  • img: applies rule to image
  • display:block = assigns each image to start on a new line
  • when we create an externall CSS file, we need to link it to the HTML file; use the HTML link element
  • link element creates the connection to the external file
  • rel attribute specifies the relatinoship between the current doc and linked doc
  • href attribute specifies the location (URL) of the external resource
  • .card designates a class selector; any HTML element assigned to the class will gain those property assignments
  • can assign colors using semantic references (e.g. gray or blue) or by a hexadecimal code or hex #
  • always need to link the CSS file to the HTML file within the head element

Git

  • git is a version control system that lets you manage & keep track of each version of your project
  • github is a cloud-based hosting service that allows you to keep track of your projects inside of repositories (repos) that are stored remotely
  • repositories (repos): folders in GitHub that store files that contain code, files that contain other information about the code, like README file, and the version history for all of the files in the repo
  • when creating a new repository in GitHub, use a name that is unique and descriptive, lowercase letters, dashes not spaces, and consistent
  • README files are used to pass along useful information to anyone who visits the repo on GitHub
  • licenses allow developers to use your code fairly; MIT license allows for commercial use
  • use "Issues" to note the user story and acceptance crietria
  • the main branch is the default branch; you should create a feature branch which is a copy of the main branch to make changes
  • in the feature branch, you make commits of all changes made to the code with a commit message that is descriptive of what changed in the code; it's important to commit often
  • a pull request (PR) is used to merge a feature branch into the main branch
  • git commands in the command line always start with git
  • git clone [paste copied SSH value]: clones a GitHub repo onto your computer
  • git status: checks what branch we are currently on
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git add -A: command adds modifications in the current working branch to the staging area
  • git commit -m [message that describes what the commit contains]: creates a commit that captures a snapshot of the currently staged changes; key to writing effective commit messages is to write them as imperative commands in the present tense
  • git pull origin main: pulls down any changes from the base branch to double-check that the local branch is in sync with the base branch
  • git push origin [branch-name]: pushes changes to the remote branch
  • git revert [SHA]: reverts to a previous version history
  • SHA, or hash: a unique identifier that allows you to keep a record of the specific changes committed; devs often only type out the first seven numbers or letters
  • see git-cheat-sheet for more common git commands

JavaScript

  • JavaScript (JS) creates logic & allows the user to interact with the application
  • see javascript basics and javascript data types for an overview of JS
  • JS variable (var): a named container that allows you to store data in your code
  • assignment operator = assigns specific data that you want stored in your variable
  • e.g. var shape = circle; shape is the name of the variable and circle is the data it stores
  • a variable can contain a string that includes all topics within quotation marks
  • console.log method ouputs a message to the web console by adding an argument; see more on console.log here
  • always need to link the JS file to the HTML file within the body element
  • can use Chrome DevTools to view console.log output; this method makes it easier for devs to test their code and debug
  • in JS, control flow code runs from the first statement at the top of the page down to the last statement at the bottom of the page
  • conditional statements and loops control the order in which code is executed
  • an if statement is a way of introducing decision-making into your code
  • when a computer encounters an if statement, it will check to see if the condition provided is truthy - a Boolean with a value of true
  • === is a strict equality operator that checks to see if two values are equal
  • array: a single variable that is used to hold a group of data; string data needs to be contained within quotation marks unlike numerical or Boolean values
  • index: a unique number that tells you the location of the data item in the array; starts at 0
  • for loop: a type of loop that uses the predictable patter of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • first statement in a for loop determines the starting point for the loop
  • the second statement in a for loop is the condition
  • the final statement in a for loop allows the array to iterate over each item
  • commenting out code with /* */ makes it inactive without deleting it
  • a function is a set of instructions that tells the computer how to perform a certain task; a function's name is folloowed by parentheses
  • use a math function to randomize the selection

Vocabulary

  • user story: a short description of the user you're creating an application for (type of person using the application, what the user wants from the application, why the user wants what they want)
  • acceptance criteria: list of the things your application must achieve in order to be considered done
  • computational thinking: process of understanding a complex problem by breaking that problem into smaller parts & then developing possible solutions that can be presented in an understandable and clear way
  • 4 cornerstones of computational thinking: decomposition (breaking down a problem into smaller tasks), pattern recognition (finding patterns to solve a particular problem), abstraction (focusing only on relevant information), & testing and debugging algorithms
  • event trigger: when the completion of the first event triggers the second event
  • conditional statement: allows you to make decision based on conditions in your code
  • debugging: changing the algorithm after finding a mistake in the code
  • loop: an action that will repeat over and over again until a condition is met
  • agile: a framework for how to deal with issues that focuses on the people doing the work and how they work together
  • terminal: the application that allows you to interact with the shell
  • package: contains code written by others that you can add to and use on your computer and in projects
  • pair programming: when two developers collaborate on a feature or a project
  • driver-navigator model: a common pair programming strategy with a developer using the keyboard to type code (driver) and another developer who is responsible for the overall direction of the project (navigator)
  • web development project: a web application or web app that provides a solution to a real-world problem
  • web application: an application, accessed via a web browser that allows users to perform tasks on the internet
  • uniform resource locator (URL): the unique address given to a specific resource on the web
  • minimum viable produce (MVP): first iteration of a product without extra features; an important milestone in the development process because it fulfills the requirements and proves that the solution to the program works
  • Markdown: a lightweight plain-text formatting language similar to working with a .doc
  • blocker/bottleneck: stuck on a particular problem or a point in the development process that is slowing down the rest of production
  • feature: a functional part of the application or website that has value or fulfills a need in the business
  • graphical user interface (GUI): a way for users to interact with the computer visually
  • separation of concerns: separating code into different files based on use; a key design principle to ensure that each section of code has its own responsibility
  • Do Not Repeaty Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained; greatly reduces the overal size and complexity of a codebase
  • control flow: the order in which a computer executes code in a script
  • boolean data type: has one of two possible values (true or false) that represent the two truth values of logic
